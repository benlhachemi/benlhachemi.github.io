## 🧠 My Engineering Philosophy
- **Build → Iterate → Loop:** I start with implementing ideas as simple as possible, get it to work first with minimal setup, then iterate and improve.

- **Ship > Perfect:** A deployed, working product is infinitely better than a perfect, unfinished one.

- **I don't overplan:** I start with a clear mental model, choose the right tools, and ship fast.

- **UI/UX is a Feature:** Micro-interactions, accessibility, and polish aren't "nice-to-haves." They are the product.

- **End-to-End Ownership:** I don't just write the frontend. I care about the database schema, the API design, and the deployment pipeline.

- **Learning by doing:** I prefer building real products over tutorial projects and learning through implementation rather than endless courses.

## 🔁 My Workflow

1. Understand user problem
2. Figure out all possible solutions 
3. Analyze and compare those solutions by cost, feasibility and technical complexity
4. Build & Ship MVP
5. Measure usage, traffic and runtime errors
6. Iterate from feedback and metrics
